You are working in the team repo for the **CSE 240D "Fast Enough to Act"** final project — a roofline study comparing OpenVLA (autoregressive VLA, memory-bandwidth-bound) against Diffusion Policy (iterative denoising, compute-bound) on a fixed RTX 4060 Laptop GPU. ## One-sentence goal

Plot both paradigms' dominant kernels on the **same roofline**, then measure how each paradigm-specific and shared optimization (INT-quant, FA-2, DDIM step reduction, torch.compile, ...) moves the dot — showing the two workloads respond *differently* because they start on opposite sides of the ridge.

## Rules for any agent acting in this repo

1. **Never push results into `results/`.** That dir is `.gitkeep` only — real artifacts (.nsys-rep, .ncu-rep, CSV, PNG) go to `/srv/240d/results/` on cortex. Accidentally committing a 400 MB `.ncu-rep` to git is unrecoverable — the `.gitignore` protects against some but not all extensions.

2. **Changes to `src/` affect all three teammates.** `src/timing.py` and `src/plot_roofline.py` are shared. Don't refactor aggressively without flagging a PR discussion. Bug fixes and small extensions are fine.

3. **Per-role envs are separate on purpose.** Don't collapse `openvla.yml` / `diffusion.yml` / `profiling.yml` into one big env. PyTorch + CUDA versions need to be uniform (all 2.4.1 + CUDA 12.4), but the packages layered on top differ and we want per-role reproducibility.

4. **GPU-using scripts must assume exclusive access.** `bench_openvla.py`, `bench_dp.py`, and anything that calls `torch.cuda.Event` should be run on cortex *after* the user has called `240d-gpu claim`. If you're editing one of these scripts, don't add subtle background GPU work (preloading, warmup threads) that might interfere with the teammate who has the GPU.

5. **Default timing is `torch.cuda.Event` with warmup + repeats.** `src/timing.py:benchmark()` is the canonical helper — warmup N iterations, time M, report (mean, median, p95, std). Don't introduce `time.time()` or `timeit` for CUDA work.

6. **No AI attribution anywhere.** Don't add "Co-Authored-By Claude" to commits. Don't leave "generated by" comments in code. Academic integrity — this is a graded project. Universal rule from `~/.ai/rules/always/behavioral.md`.

7. **Before writing code that touches a paradigm you don't own**, ask. Purush owns OpenVLA, Yuva owns Diffusion Policy, Ethan owns Infra. Overlaps happen (e.g. plotting helpers), but cross-role changes are PR-discussed, not silently made.

8. **Python style: match what's in `src/timing.py` and `src/plot_roofline.py`.** Type hints on function signatures, docstrings in the Feynman-explain-the-why style (see `~/.ai/rules/contextual/vault-writing.md`), no unnecessary abstractions.

## Running the scripts

On cortex (after `240d-gpu claim`):

```bash
# Purush's role
conda activate openvla
python scripts/bench_openvla.py --runs 10 --warmup 3 \
    --out /srv/240d/results/latency/purush/$(date +%F)/openvla-baseline.json

# Yuva's role
conda activate diffusion
python scripts/bench_dp.py --ddim-steps 10 --runs 10 --warmup 3 \
    --out /srv/240d/results/latency/yuva/$(date +%F)/dp-ddim10.json

# Ethan's role (plotting)
conda activate profiling
python src/plot_roofline.py \
    --latency /srv/240d/results/latency/purush/$(date +%F)/openvla-baseline.json \
    --ncu /srv/240d/results/ncu/purush/$(date +%F)/fa2-fwd.csv \
    --out /srv/240d/results/roofline/ethan/$(date +%F)/paired-paradigm.png
```
